# Contributing Guidelines

Thank you for your interest in contributing to this project! To ensure a smooth collaboration and maintain project quality, please follow the guidelines below.

This project is maintained by **Raihan Marwanda ([@sijawiir](https://github.com/sijawiir))**.

---

## 🛠️ How to Contribute

We follow a standard **Fork & Pull Request** workflow. Please follow these steps:

### 1. Fork & Clone
* **Fork** this repository to your own GitHub account.
* **Clone** your fork to your local machine:
    ```bash
    git clone [https://github.com/your-username/repository-name.git](https://github.com/your-username/repository-name.git)
    ```

### 2. Setup Environment
* Initialize the project based on the documentation provided in the `README.md`.
* Ensure you have all the necessary tools and dependencies installed.
* Create a `.env` file if the project requires environment variables.

### 3. Create a Branch
* Always create a new branch for your work. Use a descriptive name:
    ```bash
    git checkout -b feature/amazing-feature
    # or
    git checkout -b fix/issue-description
    ```

### 4. Commit & Push
* Write clean, readable, and well-documented code.
* Use descriptive commit messages:
    ```bash
    git commit -m "feat: add user authentication"
    ```
* Push your branch to your fork:
    ```bash
    git push origin your-branch-name
    ```

### 5. Open a Pull Request
* Open a **Pull Request** (PR) from your fork to our `main` branch.
* Provide a clear summary of your changes and reference any related issues.

---

## 📏 Standards & Expectations

* **Code Quality**: Follow the existing code style and architecture of the project.
* **Documentation**: If you add new functionality, please update the documentation accordingly.
* **Respect**: Adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) in all community interactions.
* **Review**: All PRs will be manually reviewed by the maintainer. We may request further modifications before merging.
